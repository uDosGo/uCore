description: Detect and fix broken file-splitter outputs that have missing type
  imports, undefined references, or duplicated content
---

When encountering TypeScript errors in files split by the file-splitter tool (detected by "Auto-generated by file-splitter" comment):

1. IDENTIFY broken splits by running `npx tsc --noEmit | grep -E "src/" | sed 's/.tsx.*/.tsx/' | sed 's/.*src/src/' | sort -u` to get unique files with errors.

2. For each broken_sibling.tsx that has missing type references (`Cannot find name 'X'`), check if the type is defined in a file-splitter generated sibling. If so, move the type definitions to a shared `types.ts` file in the same directory, and import them in all split files.

3. For files with duplicated content (two copies of the same code, first copy has missing closing braces on interfaces), strip the broken first copy. The boundary is typically at the second occurrence of `// ─── Constants` or the second `interface` definition. Use Python to find the exact split point:
   ```python
   with open(filepath) as f:
       lines = f.readlines()
   # Find second occurrence of key marker
   marker = "// ─── Constants"
   idx = [i for i, l in enumerate(lines) if marker in l]
   if len(idx) >= 2:
       with open(filepath, 'w') as f:
           f.writelines(lines[idx[1]:])
   ```

4. For files that were split but have a backup at `/Users/fredbook/.snackbar/backups/file-splitter/`, restore the original single-file version and remove the split files if the original was under the threshold.

5. After fixing, run `npx tsc --noEmit` to verify zero errors from that file.
